# Admin (PHP) Dokumen ini mencakup panduan untuk pengembang internal dan ringkasan panduan end user (operator/admin bisnis). - Untuk peta proyek lengkap, lihat ../README.md - Untuk API yang digunakan oleh aplikasi User, lihat ../docs/API.md ## Prasyarat (Dev) - PHP vX.Y - Composer - Database (MySQL/PostgreSQL) - Ekstensi PHP yang diperlukan (sesuaikan framework) ## Setup (Dev) `bash cp .env.example .env composer install # Jika Laravel: php artisan key:generate # Migrasi & seeding (opsional): php artisan migrate --seed # Jalankan server dev: php artisan serve ` Jika menggunakan framework lain (CodeIgniter, dsb.), sesuaikan perintah di atas. Detail lebih lanjut dapat ditambahkan oleh maintainer. ## Konfigurasi (Dev) - DB: DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD - APP: APP_NAME, APP_ENV, APP_URL - Auth: (mis. JWT/Session) — pastikan konsisten dengan docs/API.md ## Struktur Folder (Contoh Umum) ` admin/ app/ public/ resources/ routes/ config/ ... ` ## Testing & Linting (Dev) `bash # Contoh Laravel/PHPUnit composer test # Static analysis (opsional) # vendor/bin/phpstan analyse ` ## Deployment (Ringkas) - Build artefak jika perlu - Jalankan migrasi DB pada lingkungan produksi - Set variabel lingkungan sesuai .env ## Panduan End User Admin Bagian ini untuk operator/admin non-teknis yang mengelola panel. ### Login - Buka URL panel admin: https://{domain}/admin - Masukkan username dan password yang diberikan ### Navigasi Utama - Dashboard: ringkasan metrik - Data Master: kelola data utama - Transaksi/Operasional: proses harian - Pengaturan: manajemen pengguna & hak akses ### Tugas Umum - Menambah data: klik tombol "Tambah", isi form, simpan - Mengubah data: klik ikon edit pada baris, simpan - Menghapus data: klik ikon hapus, konfirmasi - Ekspor data: gunakan tombol ekspor (jika tersedia) ### Reset Kata Sandi - Gunakan fitur "Lupa Kata Sandi" atau hubungi admin sistem > Catatan: Detail menu/fungsi spesifik silakan lengkapi sesuai modul yang tersedia. ## Troubleshooting - Port bentrok saat serve → ubah port atau hentikan proses yang aktif - Koneksi DB gagal → periksa kredensial .env dan izin firewall - 500 Internal Error → cek log aplikasi (mis. storage/logs)
